# Learning OS 系统架构设计文档（Architecture Design v0）

---

# 一、系统总体架构

## 1.1 架构目标

系统必须满足四个核心能力：

```text id="arch-1"
① AI生成学习计划（Plan Generation）

② 结构化解析学习计划（DSL Parser）

③ 执行与打卡系统（Execution Engine）

④ 成长与激励系统（Motivation Engine）
```

---

## 1.2 总体架构图

```text id="arch-2"
        ┌────────────────────────┐
        │      AI Prompt层       │
        │ ChatGPT / Claude / LLM │
        └─────────┬──────────────┘
                  │ Markdown DSL
                  ▼
        ┌────────────────────────┐
        │   Learning OS Parser   │
        │  (Markdown → Graph)    │
        └─────────┬──────────────┘
                  ▼
        ┌────────────────────────┐
        │   Core Domain Engine   │
        │ Project / Task / XP    │
        └──────┬───────┬─────────┘
               │       │
               ▼       ▼
     ┌────────────┐  ┌────────────┐
     │ Task Engine │  │ Reward Sys │
     └────┬───────┘  └────┬───────┘
          ▼               ▼
   ┌──────────────────────────┐
   │      UI / Mini App       │
   │  (WeChat / Web / App)    │
   └──────────────────────────┘
```

# 二、核心模块拆解

---

# 2.1 AI计划生成模块（Plan Generator）

## 作用

将用户输入转化为：

> 标准 Learning OS Markdown DSL

---

## 输入

```text id="arch-3"
目标：
从0学习量化交易

周期：
180天

奖励：
Mac Studio
```

---

## 输出

标准 Markdown DSL：

```markdown id="arch-4"
# Project

# Stage 1

## Task
id: T001
title: 市场基础
type: theory
xp: 20
```

---

## AI职责

必须完成：

* 拆解目标
* 分阶段（Stage）
* 每个任务理论+实践
* 控制每日学习量
* 分配XP
* 拆解奖励

---

## 关键设计点

👉 AI只负责“设计”，不负责执行

---

# 2.2 Markdown DSL Parser（核心系统）

## 作用

把 AI 输出的 Markdown 转换为结构化数据。

---

## 输入

```markdown
Project → Stage → Task
```

---

## 输出数据结构

```json id="arch-5"
{
  "project": {},
  "stages": [
    {
      "tasks": [
        {
          "id": "T001",
          "type": "practice",
          "xp": 30,
          "status": "pending"
        }
      ]
    }
  ]
}
```

---

## 关键能力

* Markdown → Graph
* Task依赖解析
* XP计算
* 状态管理

---

# 2.3 Core Domain Engine（核心业务引擎）

---

## 3大核心对象

---

### ① Project（项目）

```text id="arch-6"
长期学习目标容器
```

例如：

* 量化交易
* CFA
* Rust
* AI工程

---

### ② Stage（阶段）

```text id="arch-7"
学习阶段
```

例如：

* 基础
* 进阶
* 项目实践

---

### ③ Task（任务）

```text id="arch-8"
最小执行单元
```

属性：

* type（theory / practice / project）
* xp
* depends
* status
* estimate

---

## 状态流转

```text id="arch-9"
pending → doing → done
```

---

# 2.4 Task Execution Engine（执行引擎）

## 作用

管理用户每天的学习行为。

---

## 输入

```text id="arch-10"
用户点击：完成打卡
```

---

## 输出

```text id="arch-11"
XP +30
任务完成
解锁下一任务
更新进度
```

---

## 关键机制

### 1. 依赖解锁

```text id="arch-12"
T002 必须完成 T001
```

---

### 2. 自动推进

完成一个 Task：

自动推荐下一个 Task

---

### 3. 时间估算控制

防止任务过重：

```text id="arch-13"
每天 ≤ 60分钟
```

---

# 2.5 Reward Engine（奖励系统）

---

## 输入

```text id="arch-14"
任务完成 / 阶段完成 / 项目完成
```

---

## 输出

```text id="arch-15"
XP + 成长值 + 梦想值
```

---

## 核心设计

### 梦想货币系统

```text id="arch-16"
1 Task = 100梦想值（映射现实奖励）
```

---

## 奖励拆解

```text id="arch-17"
Mac Studio = 15000元 = 150任务
```

---

## 激励类型

* 即时奖励（XP）
* 中期奖励（阶段完成）
* 长期奖励（梦想）

---

# 2.6 Motivation Engine（情绪引擎）

---

## 作用

让用户“想继续学”。

---

## 四种反馈

### ① 成长反馈

```text id="arch-18"
你比昨天更接近目标了
```

---

### ② 梦想反馈

```text id="arch-19"
距离Mac Studio：还差52天
```

---

### ③ 回顾反馈

```text id="arch-20"
30天前你还不会下载数据
```

---

### ④ 情绪反馈

```text id="arch-21"
今天没有完成，但你没有失败
```

# 八、核心设计原则

---

## 1. AI负责规划

系统不做“聪明决策”。

---

## 2. 系统负责执行

严格结构化。

---

## 3. 用户只做一件事：

```text id="arch-33"
今天这个Task完成了吗？
```

---

## 4. 所有复杂性都被隐藏

用户只看到：

* 今天做什么
* 是否完成
* 离梦想多远

---

# 九、系统本质总结

```text id="arch-34"
AI（规划未来）
    ↓
Markdown DSL（结构化未来）
    ↓
Learning OS（执行未来）
    ↓
用户行为（每天一点点）
    ↓
奖励系统（持续反馈）
    ↓
长期坚持
```

---

# 十、一句话架构总结

> Learning OS 是一个“AI生成学习计划 + 结构化执行引擎 + 梦想激励系统”的长期学习操作系统。
