# Learning OS MVP 技术选型与启动方案（v0）

---

# 一、MVP目标（重新定义）

## 🎯 MVP只验证一件事：

> 用户是否愿意把“AI生成的学习计划”变成长期执行行为

---

## ✔ MVP必须实现闭环

```text id="mvp2-1"
Markdown学习计划（外部AI生成）
        ↓
导入 Learning OS
        ↓
解析成任务结构
        ↓
每日任务展示
        ↓
打卡完成
        ↓
XP + 成长 + 梦想进度
```

---

## ❌ MVP不包含

```text id="mvp2-2"
❌ AI生成计划
❌ Prompt系统
❌ 模型调用
❌ 自动规划
```

---

## ✔ MVP核心能力

```text id="mvp2-3"
✔ DSL解析
✔ 任务管理
✔ 打卡系统
✔ XP系统
✔ 梦想进度系统
✔ 简单UI展示
```

---

# 二、技术选型（极简稳定版）

---

# 2.1 前端（推荐）

## ⭐ 方案：UniApp（最优MVP选择）

理由：

```text id="mvp2-4"
✔ 一次开发，多端运行（微信 + Web）
✔ 适合快速UI验证
✔ 学习成本低
✔ 国内生态成熟
```

---

## 前端结构

```text id="mvp2-5"
pages/
  home/          首页（今日任务 + 梦想）
  task/          任务详情
  map/           学习地图（DSL结构）
  reward/        梦想奖励
  profile/       用户中心
```

---

## 状态管理

```text id="mvp2-6"
Pinia / Vuex

ProjectState
TaskState
UserState
RewardState
```

---

# 2.2 后端（推荐）

## ⭐ 方案：FastAPI（Python）

理由：

```text id="mvp2-7"
✔ JSON天然适配DSL
✔ 适合任务系统
✔ 开发速度快
✔ 后期可扩展量化模块
```

---

## 后端结构

```text id="mvp2-8"
app/
  api/
    project.py
    task.py
    progress.py
  services/
    parser.py      # DSL解析核心
    engine.py      # 任务状态机
    reward.py      # XP系统
  models/
    db.py
```

---

# 2.3 数据库（MVP级）

## ⭐ 方案：SQLite（优先）

理由：

```text id="mvp2-9"
✔ 零部署成本
✔ 单用户友好
✔ MVP足够
✔ 可后期升级PostgreSQL
```

# 三、核心系统设计

---

# 3.1 Markdown DSL导入系统（核心）

## 输入

用户粘贴：

```markdown id="mvp2-13"
# Project
# Stage
## Task
```

---

## 输出

```text id="mvp2-14"
JSON任务树
```

---

## 解析方式（MVP实现）

```text id="mvp2-15"
不做AST解析

采用规则解析：

# Project → project
# Stage → stage
## Task → task
```

---

## 关键点

👉 DSL是系统唯一输入标准

---

# 3.2 Task状态机（核心）

---

## 状态定义

```text id="mvp2-16"
pending → doing → done
```

---

## 状态流转

```text id="mvp2-17"
点击开始 → doing
点击完成 → done
```

---

## 规则

```text id="mvp2-18"
完成 Task → XP + 梦想值
```

---

# 3.3 任务调度逻辑

---

## 今日任务生成

```text id="mvp2-19"
规则：

1. 未完成Task
2. 依赖已完成
3. 时间≤60分钟
4. 优先Stage顺序
```

---

# 3.4 XP与奖励系统

---

## XP规则

```text id="mvp2-20"
Task XP = 10 ~ 50
```

---

## 梦想值系统

```text id="mvp2-21"
1 XP ≈ 1 元成长价值（可配置）
```

---

## 梦想进度

```text id="mvp2-22"
完成任务 → 累计“接近梦想值”
```

# 九、产品本质（最终定义）

---

> Learning OS 是一个 AI 无关的学习执行系统，它将外部 AI 生成的 Markdown 学习计划转化为结构化任务流，并通过打卡、XP与奖励系统帮助用户持续执行长期学习目标。
